# A3 Task Card v0

## 1. 题目一句话目标
建立 A3 扩展家族的题库占位入口，供中期后接入新的非 A1 / A2 主案例。

## 2. 当前定位
- 当前状态：`placeholder`
- 本卡只用于说明 A3 已进入题库骨架，不代表已经完成 requirement、固定协议或正式实验。
- 中期阶段不应把 A3 写成“已完成主案例”。

## 3. 候选子方向
- 曲线拟合
- 昆虫分类

## 4. 预期输入 / 输出
输入：
- 待冻结的具体题目定义
- 对应数据来源、接口约束和最小评测协议

输出：
- A3 的 `requirement` 文档
- A3 的 `taskcard_v1`
- 固定协议、failure case 与 run / metrics / report 接入点

## 5. 接入前必须先冻结的内容
- 先从候选方向中确定 1 个首发条目。
- 明确输入 / 输出接口与最小正确性标准。
- 明确是否更偏“建模求解”还是“分类预测”。
- 明确固定协议与最小 failure-case 设计方式。

## 6. 中期阶段可诚实表述的状态
- A3 已完成题库占位接入。
- A3 尚未进入当前“已闭环主案例”范围。
- A3 的 requirement、协议、实验与结论仍待下一阶段补入。

## 7. 下一步最小动作
1. 在 A3 候选方向中冻结 1 个具体任务。
2. 补 `requirement` 文档。
3. 将本卡升级为 `taskcard_v1`。
