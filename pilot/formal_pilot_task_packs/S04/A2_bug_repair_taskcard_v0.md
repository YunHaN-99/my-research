# A2 Bug Repair Task Card v0

## 1. 题目一句话目标
给定 A2 `rslt_inpainting` 的失败样例，先定位主错误，再在不破坏原接口与 mask 语义的前提下做最小可运行修补。

## 2. 输入 / 输出
输入：
- `symptom.md`：外部现象、触发条件、固定协议上下文
- `buggy_code.py`：带缺陷的最小代码片段
- 原任务上下文：A2 `rslt_inpainting(observed, mask, ...)` 的目标与接口约束

输出：
- diagnosis：一句话主错误定位 + 2 到 4 行原因说明
- patched code：最小修补版本
- regression notes：说明为什么修补后不会复发同类问题
- 实验产物：run 记录 + metrics 条目

## 3. 修补约束
- 保持原函数名、参数列表和返回约定，除非 symptom 明确表明接口本身错误。
- 优先修根因，不要只压低报错或删除关键约束。
- patch 应尽量最小，避免顺手重写整段 patch-group 流程。
- 不允许把 `diagnosis.md` 或 `fixed_code.py` 当作给模型的输入材料。

## 4. 推荐工作流
1. 先重述 symptom，明确“理想行为”是什么。
2. 找到被破坏的核心不变量。
3. 只修改与主错误直接相关的代码。
4. 做一次最小回归检查，确认没有引入新的 mask / patch / RPCA 语义错误。

## 5. 典型不变量
- `mask > 0.5` 表示观测像素，必须在初始化和每轮迭代后保持与 `observed` 一致。
- 聚合阶段必须回写 RPCA 的低秩分量，而不是稀疏残差。
- 输出必须保持与输入同 shape，并在返回前 clip 到 `[0, 1]`。

## 6. 常见失败方式
- 把 `mask == 0` 当成观测像素，导致掩码语义反了。
- 聚合后忘记重新施加 `mask` 约束，观测区被改写。
- 把 `sparse` 分量当成恢复结果写回图像。
- 为了“修好”当前 case，直接改函数签名或删掉 `history` 返回。
- 用更大的重构掩盖局部 bug，导致 patch 不可替换或不可复核。
