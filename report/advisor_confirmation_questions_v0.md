# Advisor Confirmation Questions v0

## 目的
把当前最需要导师拍板的 3 个问题固定成一页可直接发送的确认清单，尽快锁定中期表述边界和中期后的优先级。

## 当前前提
- A1 / A2 已足够支撑“首轮主案例闭环”这一阶段判断。
- A2 当前口径已收束为低秩图像任务族下、已完成范围收束并完成主案例闭环的 `hw2-op2/src/chapter5_rslt.py::rslt_inpainting(...)` 灰度图像修复主案例。
- 当前更需要的是锁定导师预期，而不是继续独自追加同类 rerun。

## 可直接发导师的版本

老师您好，我想在中期材料最终收口前，先请您帮我确认 3 个关键问题，这会直接决定我接下来材料如何写、以及中期后优先往哪个方向推进。

目前我准备把中期的一句话阶段判断统一写成：

> A1 / A2 首轮主案例闭环已完成；当前中期重点是收束模板库、检查清单、错误分类、题库页和原型入口。

同时，我也准备把研究结论冻结为两句教育研究语言：

> `plain_guidance` / `coe_guided` 的核心优势，是提升自检覆盖、根因说明和回归说明的可复查性。
>
> 在 A2 这类冻结协议任务上，结构化 guidance 未必显著改变最终恢复指标，但确实改善了过程质量与可解释性。

### 1. 是否认可 A2 的范围收束
目前我准备把 A2 从早期较宽的 “SVD 图像压缩” 表述，正式收束为当前仓库中的“低秩图像任务族下、已完成范围收束并完成主案例闭环的 `hw2-op2/src/chapter5_rslt.py::rslt_inpainting(...)` 灰度图像修复主案例”，并在中期材料中写成：

> A2 已完成范围收束，并在当前冻结口径下完成主案例闭环。

想请您确认，您是否认可我按这个口径来写中期材料。

可选判断：
- A. 认可，按当前收束口径写即可。
- B. 基本认可，但需要我在中期材料里再补一段解释，说明它与原始 Timeline 的对应关系。
- C. 不认可，A2 仍应按更宽的原始标签来定义。

### 2. 中期材料更应该强调哪一类成果
目前仓库里已经有三类材料：
- 方法与流程：prompt、task card、结构化检查、bug-repair benchmark、CLI 原型。
- 量化结果：A1 / A2 的 guidance、performance、bug-repair、expanded-scope 等 metrics。
- 教学化转化：题库骨架、统一入口、后续可扩成教学演示的原型方向。

想请您判断，中期汇报更应把哪一类作为主轴。

可选判断：
- A. 以方法流程和教育研究结论为主，量化结果作为支撑。
- B. 以量化结果为主，方法流程作为辅助说明。
- C. 以“教学化转化潜力”为主，突出题库和原型方向。
- D. 需要两条线并重。

如果是 `D`，也希望您顺手指定主次顺序，例如“方法流程第一，量化结果第二”。

### 3. 中期后优先级希望我怎么排
中期之后我目前有三个可推进方向：
- A3 / A4 首发条目，继续把它们从 `requirement + taskcard_v1` 推进到固定协议和实验链路。
- 统一原型，把现有脚本链路继续封装成更完整的 CLI / Notebook 入口。
- 小范围试用，开始收集学生反馈和更贴近教学场景的数据。

想请您帮我判断，哪个方向更值得优先投入。

可选判断：
- A. 先推进 A3 / A4 首发条目。
- B. 先继续做统一原型。
- C. 先开始小范围试用。
- D. 以上三者请给一个优先级顺序。

## 建议默认执行口径
如果导师没有逐条展开，只给简短反馈，当前最适合作为默认执行口径的是：
- Q1：按当前 A2 收束口径写中期材料。
- Q2：以“方法流程 + 教育研究结论”为主轴，量化结果作为支撑，不把最终指标提升当成主结论。
- Q3：先把 A3 / A4 首发条目推进到协议和实验链路，并继续完善统一原型，小范围试用放到中期后更靠后阶段。

## 使用建议
- 这 3 问最好单独发，不要夹在长篇进度汇报里。
- 每问都给老师可直接选的判断项，降低回复成本。
- 如果老师只回复一句，也要尽量追问到“主次顺序”而不是停留在泛泛认可。
